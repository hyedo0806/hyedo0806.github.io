# BFF pattern
Backend For Frontend<br>
브라우저가 백엔드에 직접 연결하지 않고, next.js(frontend)가 중간 gateway처럼 받아서 백엔드로 전달하는 구조

- optimize performance
- reduce client-side complexity
- improve the user experience

--> 다양한 frontend application에 유용하다.(web, mobile ...)

## BFF 패턴을 사용하는 이유
- 클라이언트별 맞춤 API 제공
- frontend 로직 단순화 
- 각 BFF가 클라이언트에 필요한 데이터만 정확히 제공 --> 응답속도와 리소스 효율이 개선
- 프론트엔드별로 독립적인 개발.배포.확장이 가능
- 클라이언트별로 인증.인가 정책을 다르게 적용 --> 보안 수준 향상

## How the BFF Pattern Works?
1. 프론트엔드 분리<br>
웹, 모바일, IoT 등 각 클라이언트는 서로 다른 데이터와 기능 요구사항을 가집니다.

2. 클라이언트별 백엔드 생성<br>
각 클라이언트마다 전용 BFF를 구성하여 핵심 서비스와 프론트엔드 사이의 중개 역할을 수행합니다.

3. API 요청 흐름<br>
프론트엔드는 공통 백엔드가 아닌, 자신의 전용 BFF로 API 요청을 보냅니다.

4. 데이터 집계 및 변환<br>
BFF는 여러 내부 서비스나 외부 API로부터 데이터를 수집하고, 클라이언트에 맞게 가공합니다.

5. 최적화된 응답 제공<br>
각 클라이언트에 필요한 데이터만, 적합한 포맷과 크기로 응답합니다.

6. 프론트엔드 전용 로직 처리<br>
프론트엔드에서 처리하던 비즈니스 로직이나 변환 로직을 BFF로 이동시켜 클라이언트 복잡도를 낮춥니다.
