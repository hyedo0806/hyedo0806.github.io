# Spring Actuator
Spring Boot Actuator는 애플리케이션에 생산 환경에서 유용한 관리 및 모니터링 기능을 손쉽게 추가할 수 있는 모듈입니다. 예를 들어 헬스 체크, 메트릭 수집, 환경 정보 노출 등을 제공합니다.

## 기본적인 설정 방법
### 의존성 추가 
### 엔드포인트 노출 설정
기본적으로 /actuator/health와 /actuator/info만 활성화되어 있습니다. 다른 엔드포인트를 노출하려면 application.properties 또는 application.yml에 다음과 같이 설정할 수 있습니다:
```
management.endpoints.web.exposure.include=health,info,metrics,beans
management.endpoint.health.show-details=always
```
모든 엔드포인트를 노출하려면?
```
management.endpoints.web.exposure.include=*
```
#### 주요 엔드포인트 
| 엔드포인트                         | 설명                      |
| ----------------------------- | ----------------------- |
| `/actuator/health`            | 시스템 상태 확인 (UP/DOWN 등)   |
| `/actuator/info`              | 애플리케이션 관련 정보 표시         |
| `/actuator/metrics`           | 성능 메트릭 (JVM, HTTP 등) 조회 |
| `/actuator/env`               | 환경 구성 정보 (설정값 등)        |
| `/actuator/beans`             | 등록된 Spring beans 목록     |
| `/actuator/mappings`          | 요청 경로와 핸들러 매핑           |
| `/actuator/loggers`           | 로거 레벨 확인 및 변경           |


# Secret Manager에 응용
### 헬스 체크(Health Check)
`/actuator/health`
- OpenBao API에 연결이 가능한지
- Security Extension에 연결 가능한지
### 모니터링(Metrics)
`/actuator/metrics` 를 통해 JMX, Prometheus, OpenTelemetry 등으로 지표 수집 가능
- /secret 요청 횟수
- From Secutiry Extension HTTP 응답 상태 코드별 카운트 (200, 400, 403, 401 등)
- IpAccessFilter에서 처리된 요청/거부 비율
### Prometheus 연동
`/actuator/prometheus` 
- Prometheus가 스크래핑할 수 있는 메트릭이 노출된다.
1. JVM metrics
    - 메모리 사용량 (jvm_memory_used_bytes)
    - GC 통계 (jvm_gc_pause_seconds)
    - 스레드 수, 클래스 로딩 수 등
2. HTTP 서버 metrics
    - 요청 수 (http_server_requests_seconds_count)
    - 요청 지연 시간 (http_server_requests_seconds_sum)
    - 상태 코드별 요청 통계
3. 커스텀 metrics
    - Secret 조회 성공/실패 카운터
    - IP 검증 실패 카운터
    - 비즈니스 로직 관련 카운터/타이머/게이지
  

# Actuator 사용 방법

```
management:
  endpoints:

        include:  health, metrics, prometheus, loggers 
  endpoint:
    health:
      show-details: always
```

### custom HealthIndicator 구현
외부 API(OpenBao, Secret Service 등) 상태를 확인하고 싶은 경우 HealthIndicator 구현 
```
import org.springframework.boot.actuate.health.Health;
import org.springframework.boot.actuate.health.HealthIndicator;
import org.springframework.stereotype.Component;
import org.springframework.web.client.RestTemplate;

@Component
public class OpenBaoHealthIndicator implements HealthIndicator {

    private final RestTemplate restTemplate = new RestTemplate();

    @Override
    public Health health() {
        try {
            String url = "https://openbao.example.com/health";
            Boolean isUp = restTemplate.getForObject(url, Boolean.class);

            if (Boolean.TRUE.equals(isUp)) {
                return Health.up().withDetail("OpenBao", "Available").build();
            } else {
                return Health.down().withDetail("OpenBao", "Unavailable").build();
            }
        } catch (Exception e) {
            return Health.down(e).withDetail("OpenBao", "Error connecting").build();
        }
    }
}
```
`/actuator/health` 엔드포인트 호출 시 응답 예시:
``` 
// OpenBao가 정상일 때
"openBaoHealthIndicator": {   // 커스텀 헬스 체크
  "status": "UP",             // 애플리케이션 전반 상태가 정상
  "details": {
    "OpenBao": "Available"
  }
}

// OpenBao가 장애/응답 실패일 때
"openBaoHealthIndicator": {
  "status": "DOWN",           // 애플리케이션의 일부 컴포넌트 장애 발생
  "details": {
    "OpenBao": "Error connecting",
    "exception": "org.springframework.web.client.ResourceAccessException: I/O error on GET request..."               // 오류 메시지/예외 정보 포함
  }
}
```

## 2. 로그 기록 : logback/json --> OpenSearch에 저장

Actuator는 `/actuator/loggers` 엔드포인트에서 로그의 **내용**이 아니라 **설정**을 보여줌(어떤 패키지/클래스의 로깅 레벨이 어떤 상태인지, **런타임에 로그 레벨을 동적으로 변경 가능**)br>
로그 수집과 기록은 여전히 logback, log4j, SLF4J 등 로깅 라이브러리가 담당.<br>
### 사용 예시
예시 요청:
```
GET /actuator/loggers/com.example.worker
```
응답 예시:
```
{
  "configuredLevel": "DEBUG", // 개발자가 설정한 로깅 레벨
  "effectiveLevel": "DEBUG"   // 실제 적용되는 로깅 레벨 (상위 설정에 따라 달라질 수 있음)
}
```
실제 로그 메시지(예: Secret 조회 성공/실패, IP 검증 실패 등)는 logback, log4j 등 로깅 프레임워크가 출력<br>
Kubernetes 환경에서는 stdout/stderr → Fluent Bit/Fluentd → OpenSearch로 전송<br>
OpenSearch에서 검색/분석해야만 로그 메시지를 확인 가능<br>


## 3. 메트릭 : Micrometer/Actuator --> Prometheus + Grafana 로 시각화
Spring Boot Actuator + Micrometer + Prometheus 조합<br>
Micrometer가 자동으로 Prometheus endpoint(/actuator/prometheus)를 만들어준다.

### custom metric 등록 (예: secret API 성공/실패 카운트)
```
import io.micrometer.core.instrument.Counter;
import io.micrometer.core.instrument.MeterRegistry;
import org.springframework.stereotype.Service;

@Service
public class SecretMetricsService {

    private final Counter secretSuccessCounter;
    private final Counter secretFailureCounter;

    public SecretMetricsService(MeterRegistry registry) {
        this.secretSuccessCounter = Counter.builder("secret_requests_success_total")
                .description("Total successful secret requests")
                .register(registry);

        this.secretFailureCounter = Counter.builder("secret_requests_failure_total")
                .description("Total failed secret requests")
                .register(registry);
    }

    public void recordSuccess() {
        secretSuccessCounter.increment();
    }

    public void recordFailure() {
        secretFailureCounter.increment();
    }
}
```
### SecretController에서 활용
```
@RestController
public class SecretController {

    private final SecretAccessService secretAccessService;
    private final SecretMetricsService metricsService;

    public SecretController(SecretAccessService secretAccessService,
                            SecretMetricsService metricsService) {
        this.secretAccessService = secretAccessService;
        this.metricsService = metricsService;
    }

    @GetMapping("/secret")
    public String getSecret(@RequestParam("accountId") String accountId,
                            @RequestParam("secret") String secretPath) {
        try {
            String result = secretAccessService.fetchSecret(accountId, secretPath);
            metricsService.recordSuccess(); // ✅ 성공 카운트
            return result;
        } catch (Exception e) {
            metricsService.recordFailure(); // ❌ 실패 카운트
            throw e;
        }
    }
}
```

# Actuator의 핵심 장점
### 1. 단일 엔드포인트(/actuator)를 통한 모니터링
  - /actuator/health → 애플리케이션 및 연결된 외부 서버, DB, 메시지 큐 상태 확인

  - /actuator/metrics → JVM, HTTP, DB, 커스텀 지표 확인

  - /actuator/prometheus → Prometheus가 수집할 수 있는 형식으로 메트릭 제공
### 2. 로그와 메트릭을 동시에 활용 가능
  - 로그: 기존 log4j/logback + OpenSearch
  - 메트릭: Micrometer + Actuator → Prometheus

### 3. 외부 의존성 상태 확인
  OpenBao, Security Extension 등 연결된 시스템의 상태를 HealthIndicator로 정의 가능
