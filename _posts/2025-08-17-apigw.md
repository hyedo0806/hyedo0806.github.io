### 스테이지 생성/삭제 기록
s1: 11:10 생성 → 계속 유지<br>
s3: 11:20 생성 → 14:10 삭제 → 14:50 재생성<br>
s2: 12:40 생성 → 15:20 삭제<br>
s4: 13:10 생성 → 13:40 삭제 → 14:30 재생성<br>

### 메타데이터 테이블
컬럼: api-stage, is_delete_stage, stage_time<br>
stage_time은 스테이지 최초 생성 시각의 hour만 기록.<br>
삭제되면 is_delete_stage=TRUE<br>

### 메트릭 테이블
컬럼: api-stage-resource, call, aggregation_time<br>
특정 시점의 호출 수(call)를 기록<br>
aggregation_time 기준으로 rolling aggregation 수행<br>

### 결과 테이블(오른쪽표)
집계 시각(X:05)마다 조회 기간(1시간~multi-hour)에 대해, 살아 있는 스테이지의 호출 수(call)를 합산해서 보여줌.<br>
삭제 후 재생성된 경우, 새로운 stage_time 이후의 metric만 집계됨.<br>

## 쿼리
```
SELECT 
    m.api_stage AS stage,
    SUM(m.call) AS total_calls
FROM metric m
JOIN metadata md
  ON m.api_stage = md.api_stage
WHERE m.aggregation_time BETWEEN :start_time AND :end_time
  AND m.aggregation_time >= md.stage_time
  AND md.is_delete_stage = FALSE
GROUP BY m.api_stage;
```

### 예시 1: s1의 12:00~14:59 조회
```
SELECT 
    m.api_stage, 
    SUM(m.call) AS total_calls
FROM metric m
JOIN metadata md
  ON m.api_stage = md.api_stage
WHERE m.api_stage = 's1'
  AND m.aggregation_time BETWEEN '2025-08-17 12:00:00' AND '2025-08-17 14:59:59'
  AND m.aggregation_time >= md.stage_time
  AND md.is_delete_stage = FALSE
GROUP BY m.api_stage;
```
### 예시 2: s3의 14:00~15:59 조회
```
SELECT
```

## Metadata table
### update 규칙
1. 현재 환경에 존재하는 api-stage 가져오기<br>
  존재하는데 metadata에 없으면 → 새 row insert (delete=FALSE, stage_time=이전 시각의 hour)<br>
  존재하는데 metadata에 있고 delete=TRUE라면 → delete=FALSE 로 update, stage_time=이전 시각의 hour 갱신<br>

2. 현재 환경에는 없는데 metadata에 delete=FALSE라면<br>
  삭제된 것으로 간주 → delete=TRUE 로 update, stage_time=이전 시각의 hour 갱신<br>

### table 구조
```
-- 메타데이터
CREATE TABLE metadata (
    api_stage VARCHAR PRIMARY KEY,
    is_delete_stage BOOLEAN NOT NULL,
    stage_time TIMESTAMP NOT NULL,
    updated_at TIMESTAMP DEFAULT now()
);

-- 현재 환경에서 수집한 api-stage 임시 테이블 (매시 05분에 갱신됨)
CREATE TABLE current_stage (
    api_stage VARCHAR PRIMARY KEY
);
```

### 1. 신규 생성된 api-stage → INSERT
```
INSERT INTO metadata (api_stage, is_delete_stage, stage_time, updated_at)
SELECT c.api_stage, FALSE, date_trunc('hour', now() - interval '1 hour'), now()
FROM current_stage c
LEFT JOIN metadata m ON c.api_stage = m.api_stage
WHERE m.api_stage IS NULL;
```

### 2. 기존에 delete=TRUE 였는데 다시 생성된 api-stage → UPDATE
```
UPDATE metadata m
SET is_delete_stage = FALSE,
    stage_time = date_trunc('hour', now() - interval '1 hour'),
    updated_at = now()
FROM current_stage c
WHERE m.api_stage = c.api_stage
  AND m.is_delete_stage = TRUE;
```

### 3. 현재 존재하지 않는데 metadata에서는 delete=FALSE → UPDATE (삭제 처리)
```
UPDATE metadata m
SET is_delete_stage = TRUE,
    stage_time = date_trunc('hour', now() - interval '1 hour'),
    updated_at = now()
WHERE m.is_delete_stage = FALSE
  AND NOT EXISTS (
      SELECT 1 FROM current_stage c WHERE c.api_stage = m.api_stage
  );
```
